# Security Policy

## Supported Versions

The below mentioned versions are supported:

| Version | Supported          |
| ------- | ------------------ |
| 5.1.x   | :white_check_mark: |
| 5.0.x   | :x:                |
| 4.0.x   | :white_check_mark: |
| < 4.0   | :x:                |

## Reporting a Vulnerability

if you discover a vulnerability in ProjectPELE, please follow the below mentioned steps:

## Where to report:

email me at mosamofokeng012@gmail.com with the subject line [security] vulnerability report.

## What to include:

~A description of the vulnerability
~Steps to reproduce the issue 
~The affected version(s)
~Any potential impact or exploit scenarios 

## what to expect:

~You will recieve an acknowledgement email within 48 hours 
~We aim to provide a status update every 7 days
~If accepted a patch will be prioritized aand you'll be credited(if you so wish to do so) 
~If declined we will give a detailed explanation as to ehy that is.

## Please do not open a public github issue for security vulnerabilities as this could put other users at risk before a fix is available 

we appreciate your help in keeping ProjectPELE secure.
